Universal Card System

Unificación de simple-card, standard-card y complementary-card

✅ Beneficios de la Unificación:

  • 🎯 Una sola clase para 3 casos de uso diferentes
  • 🎨 Iconos consistentes en color primary
  • 📝 Alineación izquierda unificada
  • 📋 Listas opcionales con checkmarks
  • 🔧 Modificadores para casos específicos
  • 📱 Responsive automático

Nuestra Institución

Preuniversitario JMC se funda con la misión de contribuir con las futuras generaciones en su preparación académica y formación personal, para con ello aportar en la construcción de un proyecto de vida valioso.

Estamos convencidos de que todo ser humano cuenta con un potencial enorme para crecer, contribuir y por supuesto ser feliz. Sin embargo, para aprovechar y desarrollar ese potencial, se hace imprescindible contar con el apoyo y guía en momentos claves de la vida.

Desafío que como institución educativa hemos querido emprender, por medio de nuestro trabajo, compromiso profundo, amor por la vida y fe en el ser humano.

🔄 Plan de Migración

Cómo migrar del sistema actual al universal card system

/* ============================================ MIGRACIÓN - REEMPLAZOS DIRECTOS ============================================ */ /* 1. Simple Card → Universal Card */ .simple-card → .universal-card /* 2. Standard Card → Universal Card Extensiva */ .standard-card → .universal-card.extensive /* 3. Complementary Card → Universal Card Técnica */ .complementary-card → .universal-card.technical /* ============================================ CAMBIOS EN HTML ============================================ */ /* ANTES */ <div class="simple-card"> <div class="secondary-icon-container"> <span class="material-symbols-rounded secondary-icon">home</span> </div> ... </div> /* DESPUÉS */ <div class="universal-card"> <div class="card-icon"> <span class="material-symbols-rounded">home</span> </div> ... </div>

🎯 Pasos de Migración:

  1. Agregar el CSS de .universal-card al sistema
  2. Reemplazar clases en HTML según la tabla
  3. Cambiar .secondary-icon-container por .card-icon
  4. Eliminar .secondary-icon de los spans (automático con .card-icon)
  5. Agregar .extensive o .technical según el caso
  6. Eliminar CSS de las 3 cards obsoletas
  7. Testing responsive en todos los dispositivos
